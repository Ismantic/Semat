## Introduction

### 引言

LDA (Latent Dirichlet Allocation) 是一种经典的概率主题模型，用来挖掘文档集合中的潜在主题结构。LDA 采用两阶段的生成机制来生成文档。

**LDA 的生成过程：**
- **初始条件**：需要预设文档的主题分布 $P(z|m)$
- **生成过程**：
  - 由 $P(z|m)$ 采样主题 z
  - 由 $P(w|z)$ 采样词语 w
  - 不断重复以上步骤
- **结束条件**：达到预订的词语数量

#### 到底主题是什么

实际上，主题就是词汇表上的概率分布。每个主题代表一个语义相关的词语集合。


**示例主题：**

**主题 0（科技）：**
```
P(w|z=0) = {
  "手机": 0.12,  "苹果": 0.08,  "电脑": 0.10,
  "软件": 0.07,  "网络": 0.09,  "数据": 0.06,
  "算法": 0.05,  "程序": 0.04,  "系统": 0.03,
  "应用": 0.04,  "技术": 0.05,  ...
}
```

**主题 1（体育）：**
```
P(w|z=1) = {
  "足球": 0.15,  "比赛": 0.12,  "球员": 0.10,
  "进球": 0.08,  "教练": 0.07,  "联赛": 0.09,
  "冠军": 0.06,  "球队": 0.11,  "场地": 0.04,
  "训练": 0.05,  "战术": 0.03,  ...
}
```

**主题 2（美食）：**
```
P(w|z=2) = {
  "苹果": 0.06,  "牛肉": 0.09,  "餐厅": 0.08,
  "味道": 0.10,  "烹饪": 0.07,  "食材": 0.11,
  "美味": 0.09,  "菜谱": 0.05,  "香甜": 0.04,
  "新鲜": 0.06,  "营养": 0.05,  ...
}
```

#### 举个生成的例子

假设有这么一个文档，其主题分布为：
```
P(z|文档) = {主题0: 0.6, 主题1: 0.1, 主题2: 0.3}
```

**过程演示：**

假设有这么一个文档：
```
最新的手机配备了先进的苹果芯片，其内置软件能够高效处理海量数据。
通过智能算法和网络连接，这个程序运行流畅。
今天的比赛很精彩，现场还有美味的苹果和各种食物，味道很棒。
```

该文档包含的关键词是：`["手机", "苹果", "软件", "数据", "算法", "网络", "程序", "比赛", "美味", "味道"]`

那么这些关键词的生成过程大概是这样：

0. **第0个词**：采样主题 → 主题0（科技），采样词语 → "手机"
1. **第1个词**：采样主题 → 主题0（科技），采样词语 → "苹果"（指苹果公司/芯片）
2. **第2个词**：采样主题 → 主题0（科技），采样词语 → "软件"
3. **第3个词**：采样主题 → 主题0（科技），采样词语 → "数据"
4. **第4个词**：采样主题 → 主题0（科技），采样词语 → "算法"
5. **第5个词**：采样主题 → 主题0（科技），采样词语 → "网络"
6. **第6个词**：采样主题 → 主题0（科技），采样词语 → "程序"
7. **第7个词**：采样主题 → 主题1（体育），采样词语 → "比赛"
8. **第8个词**：采样主题 → 主题2（美食），采样词语 → "美味"
9. **第9个词**：采样主题 → 主题2（美食），采样词语 → "味道"

用这个例子说明下，概率主题模型本质上还是词袋模型，其不能像GPT一样能生成流畅的文字表达，只能生成像这样的关键词集合。(词袋，顾名思义，就是一袋子词，说明词之间没有顺序关系)

注意到，“苹果”是同时存在于两个主题中，因其是个多义词，但在这个例子中，"苹果"的主题是"科技"，这个就是LDA模型的独到之处，能解决词语的多义性，通过词语的上下文，明确词语的主题，从而能实现消歧义。

### 关键公式

LDA 模型的核心是通过 Gibbs 采用来推断每个词语的主题分配。关键的采样公式为：

$$P(z_{m,n}=t | \mathbf{z}_{-m,n}, \mathbf{w}) \propto
    \frac{(\alpha + n_{t|m}^{-m,n})(\beta + n_{w|t}^{-m,n})}
         {\sum_v(\beta + n_{v|t}^{-m,n})}$$

该公式表示：对于文档 m 中第 n 个词语，其主题分配为 t 的概率正比于右侧的表达式。

**分项解释：**
- **第一部分** $(\alpha + n_{t|m}^{-m,n})$：文档-主题倾向
  - 表示去除当前词语后，文档 m 中分配给主题 t 的词语数量
  - $\alpha$ 是平滑参数，避免零概率

- **第二部分** $(\beta + n_{w|t}^{-m,n})$：词语-主题倾向
  - 表示去除当前词语后，全部语料中词语 w 被分配给主题 t 的次数
  - $\beta$ 是平滑参数

- **分母部分** $\sum_v(\beta + n_{v|t}^{-m,n})$：归一化项
  - 表示主题 t 的总词语数量，用于归一化

**直观理解：**
假设当前词语是"苹果"，候选主题是"科技"：
- 如果当前文档中已有很多词语被分配给"科技"主题（第一部分）
- 且"苹果"在"科技"主题中出现概率高（第二部分）
- 那么"苹果"被分配给"科技"主题的概率就很高

### Sparser

标准 LDA 的采样复杂度是 O(K), 其中 K 是主题数。当主题数很大时，这会成为性能瓶颈。Sparse LDA 通过将采样概率分解为三个"桶"来解决这个问题。

#### 数学推导

由标准的 LDA 采样公式开始：

$$P(z_{m,n}=t) \propto \frac{(\alpha+n_{t|m}^{-m,n})(\beta+n_{w|t}^{-m,n})}
                            {\beta V+n_{·|t}^{-m,n}}$$

对记号简化，定义：
- $n_{t|m}$ ：文档 m 中分配给主题 t 的词数
- $n_{w|t}$ ：词语 w 被分配给主题 t 的次数
- $n_{·|t}$ ：主题 t 的总词数

**第一步：展开分子**

把分子展开：
$$(\alpha+n_{t|m})(\beta+n_{w|t}) = 
    \alpha\beta + \alpha n_{w|t} + n_{t|m}\beta + n_{t|m}n_{w|t}$$

**第二步：组织成三个部分**

把展开式重新组织：
$$\frac{\alpha\beta + \alpha n_{w|t} + n_{t|m}\beta + n_{t|m}n_{w|t}}
       {\beta V + n_{·|t}}$$

$$= \frac{\alpha\beta}{\beta V + n_{·|t}} +
    \frac{n_{t|m}\beta}{\beta V + n_{·|t}} +
    \left(\frac{\alpha n_{w|t}}{\beta V + n_{·|t}} +
    \frac{n_{t|m}n_{w|t}}{\beta V + n_{·|t}}\right)$$

**第三步：得到三个桶**

**桶 s （平滑项）：**
$$s_t = \frac{\alpha\beta}{\beta V + n_{·|t}}$$

**桶 r （文档项）：**
$$r_t = \frac{n_{t|m}\beta}{\beta V + n_{·|t}}$$

**桶 q （词语项）：**
$$q_t = \frac{\alpha n_{w|t}}{\beta V + n_{·|t}} 
        + \frac{n_{t|m}n_{w|t}}{\beta V + n_{·|t}}
      = \frac{(\alpha + n_{t|m})n_{w|t}}{\beta V + n_{·|t}}$$

用桶 s 采样的话，时间复杂度依然是 O(K)，不过用桶 r 和用桶 q 的话，时间复杂度都会小于 K ，且当 K 很大的时候，会小的很多（10 vs 1000），而且注意到桶 s 的总大小相比其它的桶其实会小很多，也就是采样会很少落到这个桶，那么其时间复杂度高也就影响不大了。

#### 具体算法

**采样步骤：**

1. **计算桶的总质量：**
$$S = \sum_{t} s_t + \sum_{t:n_{t|m}>0} r_t + \sum_{t:n_w{w|t}>0} q_t$$

2. **按比例选择桶：**
   - 随机数生成 $u \sim \text{Uniform}(0,S)$
   - 若 $u < S_s$, 选择桶 s
   - 若 $S_s \leq u < S_s + S_r$，选择桶 r
   - 否则选择桶 q
3. **桶内采样：**
   - 根据选中的桶，对应的候选主题中概率采样

注意 $S_s = \sum_{t} s_t = \sum_{t} \frac{\alpha\beta}{\beta V + n_{·|t}}$ 要缓存起来，不然整体的采样时间复杂度还是降不下来，这样的缓存会损失些精度，不过 $n_{·|t}$ 变化不大的话就还好。

### 多线程

仅仅把采样时间复杂度降下来还不够，再能通过多线程加速计算就更好了。观察Sparse LDA的采样公式，其实关键是要实现文档和词语的互斥，这样不同的线程就互相影响不到了。

**回顾下采样公式：**
$P(z_{m,n}=t) \propto \frac{(\alpha+n_{t|m})(\beta+n_{w|t})}{\beta V + n_{·|t}}$

**计数依赖分析：**
- 更新文档 m 中词语 w的主题时，会影响：
  - $n_{t|m}$ : 文档m的主题计数
  - $n_{w|t}$ : 词语w的主题计数
  - $n_{·|t}$ : 主题的总计数

**竞争条件问题：**
若多个线程同时操作，会造成这些数据出现不一致：
- **同一文档的不同词语**：会竞争修改 $n_{*|m}$
- **不同文档的相同词语**：会竞争修改 $n_{w|*}$
- **任意词语**：都会竞争修改 $n_{·|*}$  (这个竞争无法避免)

传统做法是通过给这些数据加锁，但这样就会让运行效率大为下降。

#### N-Queen 方案
类似于N皇后问题中皇后之间不能相互攻击的约束，让不同线程操作的文档和词语完全不相交，就能避免这些竞争条件。

**核心思想：**
- 把文档和词汇表都分块
- 不同线程负责不同的文档-词汇块组合
- 通过巧妙的调度避免竞争

**无锁条件：**
线程 i 处理文档块 $D_i$ 和词汇块 $V_i$，线程 j 处理文档块 $D_j$ 和词汇块 $V_j$，当满足以下竞争条件时无竞争：
- $D_i ∩ D_j = ∅$（文档块不相交）
- $V_i ∩ V_j = ∅$（词汇块不相交）

**分块策略示例（4线程，8文档，8词汇）：**

**第1轮：**
```
        词0-1  词2-3  词4-5  词6-7
文档0-1  [T0]   ×     ×     ×
文档2-3   ×    [T1]   ×     ×  
文档4-5   ×     ×    [T2]   ×
文档6-7   ×     ×     ×    [T3]
```

**第2轮（列循环右移）：**
```
        词0-1  词2-3  词4-5  词6-7
文档0-1   ×     ×     ×    [T0]
文档2-3  [T1]   ×     ×     ×
文档4-5   ×    [T2]   ×     ×
文档6-7   ×     ×    [T3]   ×
```

**第3轮（继续右移）：**
```
        词0-1  词2-3  词4-5  词6-7
文档0-1   ×     ×    [T0]   ×
文档2-3   ×     ×     ×    [T1]
文档4-5  [T2]   ×     ×     ×
文档6-7   ×    [T3]   ×     ×
```

**第4轮（完成覆盖）：**
```
        词0-1  词2-3  词4-5  词6-7
文档0-1   ×    [T0]   ×     ×
文档2-3   ×     ×    [T1]   ×
文档4-5   ×     ×     ×    [T2]
文档6-7  [T3]   ×     ×     ×
```

### Perplexity
通过计算对数似然函数能评估模型是否已经收敛了。对数似然函数衡量的是当前主题分配下，观察到的词语序列的概率。

**基本思想：** LDA假设每个词语的生成分为两步：
1. 根据文档的主题分布选择一个主题 t
2. 根据主题 t 的词语分布选择一个词语 w

**数学表达：** 对于文档 m 中位置 i 的词语 w ，其生成概率为： $P(w_{m,i})=P(t_{m,i}|m)\times P(w_{m,i}|t_{m,i})$

其中：
- $P(t|m) = \frac{n_{t|m}+\alpha}{n_m + K\alpha}$ : 文档 m 中主题 t 的概率
- $P(w|t) = \frac{n_{w|t}+\beta}{n_t + V\beta}$ : 主题 t 中词语 w 的概率

**对数似然函数：** $\log P(\text{全部词语}) = 
  \sum_{m=1}^{M}\sum_{i=1}^{N_m}[\log P(t_{m,i}|m)+\log P(w_{m,i}|t_{m,i})]$

**困惑度（Perplexity）** 定义：

$$\text{Perplexity} = \exp\left(-\frac{\log P(\text{全部词语})}{\text{总词数}}\right)$$

其可以理解为“平均每个词语有多少个等概率的候选”，数值低表示模型越好。

注意：当前这个实现不是严格意义上的Likelihood函数，只是用来评价当前训练是否收敛来用。
